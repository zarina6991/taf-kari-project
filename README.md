# Test Automation Framework (TAF) Kari Project

Репозиторий содержит каркас автоматизации тестирования (TAF) для проверки API и UI функционала интернет-магазина обуви и аксессуаров **Kari**.

---

## Стек технологий

* **Язык программирования:** Java 21
* **Сборщик проектов:** Maven
* **Тестовый фреймворк:** JUnit 5 (JUnit Platform)
* **Тестирование UI:** Selenium WebDriver (Chrome)
* **Тестирование API:** Rest Assured
* **Генерация данных:** Datafaker
* **Логирование:** SLF4J / Log4j
* **Отчетность:** Allure Report
* **Continuous Integration:** Jenkins

---

## Покрытие тестами (Текущий статус)

### API Тесты (`src/test/java/kari/api/`)
* **Login API (Negative/Positive):** Авторизация с валидными/невалидными данными, проверка обработки пустых полей и неверных форматов.
* **Recovery API:** Проверка сценариев восстановления доступа и прав авторизации.
* **Search API:** Тестирование работы поискового движка и поисковых запросов.

### UI Тесты (`src/test/java/kari/ui/`)
* **LoginTest:** Визуальная проверка формы входа и сценариев авторизации через браузер.
* **RecoveryTest:** Проверка интерфейса восстановления пароля.
* **RegistrationTest:** Тестирование сценариев регистрации (ввод неполного номера телефона).
* **SearchTest:** Поиск существующих, несуществующих товаров и проверка подсказок.

## Особенности архитектуры

### Паттерны проектирования и генерация данных
* **Page Object Model (POM):** Логика страниц сайта полностью отделена от самих тестов. Все локаторы (XPath) и действия с элементами инкапсулированы внутри классов страниц (`pages`).
* **Динамические данные:** Библиотека Datafaker используется для генерации случайных email и номеров телефонов, что исключает хардкод при проверке регистрации.

### Управление драйвером (WebDriverManager)
В проекте реализован синглтон-класс `WebDriverManager` с автоматическим разделением сред запуска:
* **Локальный запуск (IDE):** Открывает окно браузера Chrome на весь экран для удобной визуальной отладки и написания тестов.
* **Запуск на CI (Jenkins):** Автоматически определяет переменную окружения `JENKINS_URL` и переключает Chrome в скрытый режим (`--headless=new`). Это предотвращает зависание сборки в изолированной сессии `SYSTEM`.

## Запуск тестов

### Локальный запуск всех тестов:
```bash
mvn clean test
```

### Генерация и просмотр отчета Allure локально:
```bash
allure serve target/allure-results
```

---

## Интеграция с CI/CD (Jenkins)

Проект полностью настроен для работы в Jenkins на операционной системе Windows:
1. Выполнение сборки и тестов через Maven-цели: `clean test`.
2. Автоматическая проброска переменных окружения (`JENKINS_URL`) через плагин `maven-surefire-plugin`.
3. Сбор результатов тестирования и генерация интерактивных отчетов плагином **Allure Jenkins Plugin**.
